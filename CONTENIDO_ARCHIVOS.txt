FLUJO DE CARGA DE LA APLICACIÓN
================================

1. Navegador carga index.html
   ↓
2. index.html carga:
   - CSS: styles.css y pdf-viewer.css
   - JS: app.js, pdf-handler.js, all-scripts.js
   ↓
3. Se ejecuta DOMContentLoaded en app.js
   - Carga tema guardado
   - Carga datos de localStorage
   - Genera estructura de páginas
   - Inicializa componentes
   ↓
4. Usuario interactúa con la app
   - Navega por páginas
   - Crea notas
   - Carga PDFs
   - Los datos se guardan en localStorage
   ↓
5. Todo se procesa localmente (sin servidor)

CÓMO FUNCIONAN LAS NOTAS CON PDF
=================================

Flujo paso a paso:

Usuario crea nota:
    app.js: openCreateNoteModal() 
         ↓
         Abre modal con:
         - Input de título
         - Textarea de contenido
         - Área de carga de PDF
         ↓
Usuario carga PDF:
    pdf-handler.js: initPdfUpload()
                  → handlePdfDrop() O openPdfFilePicker()
                  → loadPdfFile()
                  → loadPdfFromArrayBuffer()
                  → openPdfViewer()
                  → renderAllPdfPages()
         ↓
         El PDF se visualiza en el modal
         Usuario puede hacer zoom
         ↓
Usuario guarda nota:
    app.js: saveNote()
         ↓
         pdf-handler.js: addPdfToNote()
                      → saveNoteAttachment()
         ↓
         Los datos se guardan en localStorage
         ↓
Próxima vez que abre nota:
    app.js: loadNotesUI()
         ↓
         pdf-handler.js: displayNoteAttachment()
         ↓
         El adjunto PDF aparece como botón
         ↓
Usuario hace click en adjunto:
    pdf-handler.js: viewPdfAttachment()
                  → openPdfViewer()
                  → loadPdfFromArrayBuffer()
         ↓
         PDF se abre nuevamente para ver


VARIABLES Y DATOS GUARDADOS
============================

En localStorage se guarda:

academia_theme
  → Valor: "dark" o "light"
  → Uso: Recordar preferencia de tema

academia_sounds  
  → Valor: "true" o "false"
  → Uso: Recordar si está activado sonido

academia_notes
  → Estructura: { noteId: { id, title, content, attachments } }
  → Uso: Almacenar todas las notas creadas

academia_semestres
  → Estructura: [{ id, nombre, inicio, fin, credits }]
  → Uso: Guardar semestres

academia_materias
  → Estructura: [{ id, nombre, profesor, creditos, calificacion }]
  → Uso: Guardar materias registradas

academia_tareas
  → Estructura: [{ id, titulo, fecha, prioridad, completada }]
  → Uso: Guardar tareas

En IndexedDB se guarda:

academia_images
  → Almacena imágenes grandes en base64
  → Uso: No cargar imágenes en localStorage que es limitado


SEGURIDAD Y PRIVACIDAD
======================

✅ Ventajas de almacenamiento local:
   - No se envía a servidores
   - Totalmente privado
   - Funciona sin internet (después de cargar)
   - Velocidad (no hay latencia de red)

⚠️ Limitaciones:
   - localStorage: 5-10 MB por dominio
   - IndexedDB: 50+ MB (depende del navegador)
   - Si limpias caché → Pierdes datos
   - No sincroniza entre dispositivos
   - No se guarda backup automáticamente

💡 Recomendaciones:
   - Haz backup de tus notas regularmente
   - No uses navegación privada
   - Usa siempre el mismo navegador
   - Considera exportar datos importantes
